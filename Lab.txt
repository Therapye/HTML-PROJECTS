Опис виконаних кроків:

Встановлено середовище Node.js для роботи з npm.

Створено новий проєкт за допомогою команди npm create vue@latest з назвою vue-lab.

При створенні обрано конфігурацію: TypeScript (Yes), Vue Router (Yes), Pinia (No), ESLint/Prettier (Yes).

Проаналізовано структуру проєкту: вхідну точку main.ts, головний компонент App.vue та налаштування маршрутизатора в router/index.ts.

Створено нову сторінку ContactView.vue у папці views.

Налаштовано маршрутизацію: у файл router/index.ts додано новий шлях /contact, який веде на компонент ContactView.

Оновлено навігацію в App.vue: додано посилання <RouterLink to="/contact"> для переходу на нову сторінку.

Створено компонент UserCard.vue у папці components.

У компоненті UserCard використано TypeScript (defineProps) для суворого визначення типів вхідних параметрів (props): username (рядок) та isActive (булеве значення).

Компонент UserCard інтегровано у сторінку ContactView.vue, передано відповідні дані.

Текст помилки:
При спробі передати число (наприклад, 123) у пропс isActive, який очікує boolean, TypeScript видає наступну помилку:

"Type 'number' is not assignable to type 'boolean'."

(Це означає: Тип "число" не може бути присвоєний типу "логічне значення").

Висновки:

Vue Router дозволяє створювати SPA (Single Page Applications)  веб-застосунки, які завантажуються один раз і динамічно підміняють контент при переході між сторінками. Це робить навігацію миттєвою та приємною для користувача, без повного перезавантаження сторінки.

TypeScript забезпечує статичну типізацію коду. Це дозволяє виявляти помилки ще на етапі написання коду (наприклад, якщо ми передаємо невірний тип даних у компонент), а не під час роботи програми у користувача. Це значно підвищує надійність коду та полегшує його підтримку, особливо у великих проєктах.
